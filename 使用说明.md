# 📱 SmartProperty v3.0 — 使用说明

## 🚀 快速启动（多端共同访问）

### 方式一：双击启动（推荐）
1. 双击 **`启动服务器.bat`**
2. 控制台显示服务地址后，打开浏览器
3. **手机** 或 **其他电脑** 连同一 WiFi，输入局域网地址即可

### 方式二：命令行启动
```bash
cd property-repair-system
node server.js
# 或自定义端口
PORT=8080 node server.js
```

---

## 📡 访问地址

| 设备 | 地址 |
|------|------|
| 本机浏览器 | `http://localhost:3000` |
| 手机 / 其他电脑 | `http://192.168.x.x:3000` （启动时控制台显示）|

> 💡 确保手机和电脑在同一 WiFi 网络下

## 🔌 工作模式

系统支持**两种工作模式**，自动切换：

| 模式 | 触发条件 | 数据存储 | 实时同步 |
|------|----------|----------|----------|
| **服务器模式** | 通过 `node server.js` 启动 | `data/database.json` | ✅ WebSocket 实时推送 |
| **本地文件模式** | 直接双击 `index.html` | 浏览器 localStorage | ❌ 仅当前浏览器 |

侧边栏底部状态指示器：
- 🟢 **实时同步** — 已连接服务器，多端同步生效
- 💾 **本地模式** — 服务器未启动，数据存本地

## 🔧 REST API 接口（预留扩展）

所有接口均以 `/api` 开头，方便以后对接 App 或第三方系统：

| 方法 | 接口 | 说明 |
|------|------|------|
| GET | `/api/status` | 服务器状态 |
| GET | `/api/data` | 完整数据快照 |
| POST | `/api/data` | 全量保存 |
| GET/POST/PUT/DELETE | `/api/orders` | 工单 CRUD |
| POST | `/api/orders/:id/claim` | 工程人员接单 |
| POST | `/api/orders/:id/dispatch` | 派单 |
| POST | `/api/orders/:id/complete` | 完工 |
| GET/POST/PUT/DELETE | `/api/users` | 用户管理 |
| GET/POST/PUT/DELETE | `/api/assets` | 设备资产 |
| GET/POST/PUT/DELETE | `/api/inventory` | 库存物料 |
| POST | `/api/inventory/transaction` | 出入库操作 |
| GET/POST/PUT/DELETE | `/api/fixed-assets` | 固定资产 |
| GET/POST/PUT/DELETE | `/api/technicians` | 技术人员 |
| GET/POST/PUT/DELETE | `/api/knowledge` | 知识库 |
| GET | `/api/backup` | 下载备份 |
| POST | `/api/restore` | 恢复备份 |
| GET/PUT | `/api/config` | 系统配置 |

## ⚡ WebSocket 实时事件

前端通过 WebSocket 与服务器保持长连接，以下事件触发自动刷新：

- `order_created / order_updated / order_deleted` — 工单变化
- `order_claimed / order_dispatched / order_completed` — 工单状态变化
- `inventory_changed / inventory_transaction` — 库存变化
- `asset_created / asset_updated` — 资产变化
- `data_updated / full_sync` — 全量同步

## 🔮 后续扩展建议

1. **数据库升级**：将 `database.json` 换成 SQLite / MySQL（修改 `server.js` 中的读写函数即可）
2. **用户认证**：在 `/api/*` 路由添加 JWT Token 验证
3. **HTTPS**：申请 SSL 证书，外网安全访问
4. **微信小程序对接**：调用 `/api/*` REST 接口即可
5. **推送通知**：在 `broadcast()` 函数中添加企业微信/钉钉 Webhook
